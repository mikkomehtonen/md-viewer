# Product

## Register

product

## Users

Developers and technical writers who need to quickly view markdown and text files from a local directory in a browser. They're typically in a terminal or IDE, want to check rendered output without leaving their workflow, and value speed over features.

## Product Purpose

A lightweight, zero-config file viewer that renders markdown as styled HTML and displays text files with syntax highlighting. Success means: open a directory, see your files, read them clearly. No setup friction, no cognitive overhead.

## Brand Personality

Calm, precise, efficient. The interface recedes; the content is the focus. No decoration that doesn't serve readability. Every element earns its place.

## Anti-references

- Retro/terminal aesthetics (green-on-black, monospace-everything, hacker vibes). This is a modern reading tool, not a terminal emulator.
- Heavy IDEs with chrome overload. No panel-dense, feature-overwhelm interfaces.

## Design Principles

1. **Content-first**: The file content is the product. Navigation and chrome exist only to get you to content faster.
2. **Earned simplicity**: Nothing hidden behind a menu that could be visible. Nothing visible that doesn't need to be.
3. **Quiet confidence**: No visual shouting. The design is competent without announcing itself.
4. **Native feel**: Leverage system fonts, standard patterns, and familiar interactions. Users shouldn't need to learn this interface.

## Accessibility & Inclusion

Follow best practices: semantic HTML, sufficient contrast, keyboard navigation. No formal WCAG compliance target, but no avoidable barriers either.
